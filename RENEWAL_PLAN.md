# WEEKLY-NEXO 리뉴얼 계획 (안정화용 사전 결정)

리뉴얼 본격 진행 전에 **정해 둔 결정사항**을 문서로 고정합니다.  
이 문서를 기준으로 구현하면 하위 호환과 범위 creep을 막을 수 있습니다.

---

## 🎯 리뉴얼 목표 (요약)

1. **데이터 확장**: 칼럼/기사(`articles`), 도구·자료(`tools`) 스키마 추가
2. **디자인 리뉴얼**: 모던 전자신문 + Grid + 카드 기반 UI
3. **NEXO 쌤 도구함**: 선생님용 위젯·다운로드 섹션
4. **검색**: 기본 노출 + UI 개선

---

## ✅ 사전 결정 1: 하위 호환 (Backward Compatibility)

**원칙**: 기존 발행 호에는 `articles`, `tools` 필드가 없을 수 있음.

| 상황 | 동작 |
|------|------|
| `edition.articles` 없음 또는 `[]` | "매거진(칼럼/기사)" 섹션을 **그리지 않음** |
| `edition.tools` 없음 또는 `[]` | "NEXO 쌤 도구함" 섹션을 **그리지 않음** |

**구현 시**:  
- `edition-manager.js`에서 `renderArticles()`, `renderTools()` 호출 전에  
  `if (edition.articles?.length)`, `if (edition.tools?.length)` 조건으로 감싼다.  
- 기존 `editions-data.js`의 호들은 수정하지 않아도 되며, 새 호부터만 `articles`/`tools`를 넣어도 동작하도록 한다.

---

## ✅ 사전 결정 2: 레이아웃 — Grid 우선, Masonry는 추후

**1단계(리뉴얼 본 작업)**  
- **매거진 섹션**: CSS **Grid 2~3단** 고정 컬럼으로 구현.  
- 카드 높이는 콘텐츠에 따라 자연스럽게 늘어나도 됨(높이 맞추기 X).

**이후 검토**  
- "지면처럼 높이까지 맞춘 Masonry"가 필요해지면, 그때 JS 기반 Masonry(또는 Isotope 등) 도입을 검토.

**이유**: Grid만으로도 신문 지면 느낌을 주면서 구현 부담이 적고, Masonry는 의존성·레이아웃 계산 이슈가 있어 2단계로 미룬다.

---

## ✅ 사전 결정 3: 도구함 위치 (화면 크기별)

**데스크톱 (예: 1024px 이상)**  
- **우측 사이드바** 또는 메인 콘텐츠 옆 **고정 패널**로 "NEXO 쌤 도구함" 배치.  
- 스크롤해도 도구함이 보이거나, 접기/펼치기로 노출 범위 조절.

**전자칠판·태블릿 (가로 긴 화면)**  
- 우측이 잘릴 수 있으므로, **상단 탭**(예: "기사 | 도구함") 또는 **한 줄 접기/펼치기**로 도구함 진입.  
- 또는 메인 영역 아래 별도 **풀폭 섹션**으로 배치.

**모바일**  
- **하단 시트(Sheet)** 또는 단일 컬럼에서 **섹션 블록**으로 도구함 배치.  
- 한 화면에 기사 + 도구함이 같이 보이도록 스크롤로 접근.

**구현 시**:  
- 하나의 "도구함" DOM 섹션을 두고, CSS 미디어쿼리와 클래스로 **위치/레이아웃만** 전환(우측 고정 ↔ 상단 탭/접기 ↔ 하단 섹션).  
- 동일 데이터(`tools`)로 모든 해상도에서 동작하도록 한다.

---

## ✅ 사전 결정 4: 위젯(도구) 범위 — 1단계는 모달 + 타이머 하나

**1단계 범위**  
- **모달(또는 팝업) 틀**: 도구 카드 클릭 시 모달이 열리는 공통 구조.  
- **실제 구현 위젯**: **5분 집중 타이머** 하나만 구현 (간단한 JS: 시작/일시정지/리셋).  
- 그 외 `type: 'widget'` 항목은 **카드만 노출**하고, 클릭 시 "준비 중" 또는 동일 모달 틀 + "추가 예정" 메시지로 처리.

**이후 확장**  
- 뽀모도로 타이머, 랜덤 뽑기, 간이 출석 체크 등은 2단계에서 하나씩 추가.

**이유**: 리뉴얼 범위를 넓히지 않고, "도구함 UX + 하나의 실제 도구"만 검증한 뒤 나머지는 반응을 보고 추가한다.

---

## 📂 리뉴얼 작업 순서 (참고)

1. **1단계**: 데이터 스키마 확장 (`editions-data.js` — `articles`, `tools` 추가 및 예시)
2. **2단계**: UI/UX 리뉴얼 (`index.html`, `css/style.css` — Grid, 타이포, 카드, 도구함 위치)
3. **3단계**: 렌더링 로직 (`edition-manager.js` — `renderArticles()`, `renderTools()`, 검색 노출)
4. **4단계**: 반응형·전자칠판 대응 (미디어쿼리, 도구함 위치 전환) ✅ 완료

---

## 📌 이 문서의 용도

- **리뉴얼 구현 시**: 위 4가지 사전 결정을 지키면 기존 호 깨짐 없이, 범위가 명확한 상태로 진행할 수 있음.  
- **추후 변경 시**: Masonry 도입, 위젯 추가, 도구함 위치 변경 등은 이 문서를 수정한 뒤 반영하면 됨.

리뉴얼 본격 진행 전에 이 문서를 한 번 확인하고 들어가는 것을 권장합니다.
