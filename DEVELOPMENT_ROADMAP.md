# NEXO Platform V2.0 개발 가능 여부 분석 및 로드맵

## 📊 현재 상태 분석

### 현재 기술 스택
- **프론트엔드**: HTML5, CSS3, Vanilla JavaScript (ES6+)
- **호스팅**: Netlify (정적 웹사이트)
- **백엔드**: Netlify Functions (서버리스 함수)
- **데이터 저장**: Google Sheets (주문 폼 데이터만)
- **인증**: sessionStorage 기반 간단한 관리자 비밀번호 인증

### 현재 구현된 기능
✅ 정적 콘텐츠 관리 (발행물 시스템)  
✅ Google Sheets 연동 (주문 폼 데이터 저장)  
✅ 관리자 에디터 (발행물 작성)  
✅ 반응형 디자인  
✅ 쌤 도구함 (프론트엔드 위젯)  

### 현재 제약사항
❌ 데이터베이스 없음 (Google Sheets만 사용)  
❌ 사용자 인증 시스템 없음  
❌ 회원 관리 시스템 없음  
❌ 포인트 시스템 없음  
❌ 커뮤니티 기능 없음  
❌ 알림 시스템 없음  

---

## 🎯 요구사항별 개발 가능 여부 분석

### 1. 구독 시스템 + 푸시 알림

#### 요구사항
- 회원 가입/로그인
- 매주 목요일 발행 시 핸드폰 알림톡 발송
- 구독자 관리

#### 개발 가능 여부: ⚠️ **부분 가능 (백엔드 필요)**

**현재 상태로 불가능한 이유:**
- 사용자 DB가 없음
- 알림톡 발송 API 연동 불가
- 회원 인증 시스템 없음

**필요한 기술:**
- 데이터베이스 (Supabase/Firebase 등)
- 사용자 인증 시스템 (카카오 로그인 등)
- 알림톡 API (Solapi, 알리고 등)

**단계별 구현 계획:**
1. **1단계 (즉시 가능)**: 이메일 수집 폼 (Google Forms/Tally 임베딩)
2. **2단계 (1개월)**: Supabase 도입 + 카카오 로그인 연동
3. **3단계 (2개월)**: 알림톡 API 연동 + 자동 발송 스케줄러

---

### 2. 커뮤니티 게시판 + 포인트 시스템

#### 요구사항
- 게시판 CRUD (생성/읽기/수정/삭제)
- 포인트 적립 로직 (읽기, 댓글, 후기 작성 등)
- 포인트 사용처 (경품 응모, 등급 시스템)

#### 개발 가능 여부: ❌ **현재 불가능 (백엔드 필수)**

**현재 상태로 불가능한 이유:**
- 데이터베이스 없음 (게시글 저장 불가)
- 사용자 인증 없음 (작성자 추적 불가)
- 포인트 계산/저장 시스템 없음

**필요한 기술:**
- 데이터베이스 (PostgreSQL/Supabase)
- 사용자 인증
- 포인트 로그 테이블
- 실시간 업데이트 (선택사항)

**단계별 구현 계획:**
1. **1단계 (1개월)**: Supabase 도입 + 기본 게시판 구조
2. **2단계 (2개월)**: 포인트 시스템 구현
3. **3단계 (3개월)**: 등급 시스템 + 경품 응모 기능

### 3. 구독자 전용 유료급 자료

#### 요구사항
- 비회원: "구독하고 무료 다운로드" 버튼 (블러 처리)
- 회원: 즉시 다운로드 가능
- 자료 관리 (PDF/Excel 파일)

#### 개발 가능 여부: ⚠️ **부분 가능 (인증만 추가하면 가능)**

**현재 상태:**
- 다운로드 기능은 이미 구현됨 (`assets/downloads/` 폴더)
- 인증 시스템만 추가하면 가능

**필요한 기술:**
- 사용자 인증 (Supabase Auth)
- 파일 접근 제어 (Netlify Functions로 보호)

**단계별 구현 계획:**
1. **1단계 (즉시 가능)**: 파일을 Netlify Functions 뒤에 배치하여 인증 체크
2. **2단계 (1개월)**: Supabase 로그인 연동 후 다운로드 버튼 활성화

### 4. 구독자 특가 주문 폼 + 유입 경로 추적

#### 요구사항
- 구독자 전용 특가 페이지
- 주문 신청 폼
- 유입 경로 추적 (기사님 코드, QR 코드 등)
- CRM 연동

#### 개발 가능 여부: ✅ **가능 (기존 인프라 활용)**

**현재 상태:**
- 주문 폼 이미 구현됨 (`order-form.js`)
- Google Sheets 연동 이미 구현됨
- 유입 경로 필드만 추가하면 됨

**필요한 작업:**
- 주문 폼에 `referrer_code` 필드 추가
- Google Sheets에 유입 경로 컬럼 추가
- 구독자 인증 후 특가 페이지 접근

**단계별 구현 계획:**
1. **1단계 (1주)**: 주문 폼에 유입 경로 필드 추가
2. **2단계 (2주)**: Google Sheets 스키마 확장
3. **3단계 (1개월)**: 구독자 인증 연동

### 7. 오프라인 홍보 → 온라인 유입

#### 요구사항

- QR 코드 생성 도구 (기사님별 고유 코드)
- 회원가입 폼에 `referrer_code` 필드

**단계별 구현 계획:**
1. **1단계 (1주)**: QR 코드 생성 페이지 (관리자용)

## ⚠️ 기술적 제약사항 및 해결 방안

### 제약사항 1: 정적 웹사이트의 한계
**문제**: 현재는 정적 HTML/CSS/JS만으로는 회원 시스템 불가능

**해결**: Supabase 도입 (Backend-as-a-Service)
- 무료 티어 제공
- PostgreSQL 데이터베이스
- 인증 시스템 내장
- 실시간 기능 지원

### 제약사항 2: Netlify Functions의 한계
**문제**: 서버리스 함수는 복잡한 로직에 부적합

**해결**: Supabase Edge Functions 또는 별도 백엔드
- 간단한 로직: Netlify Functions 유지
- 복잡한 로직: Supabase Functions 사용

### 제약사항 3: 파일 스토리지
**문제**: 정적 파일만 제공 가능

**해결**: Supabase Storage 또는 Netlify Large Media
- 이미지/PDF 파일 저장
- 접근 제어 가능
